# Чеклист тестирования  /pet

## POST /pet (создание питомца)

### Positive

| Тест                                                                                                                    | Ожидаемый результат               | Фактический результат                                                                                                                                           |
| ----------------------------------------------------------------------------------------------------------------------- | --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Создать питомца с обязательными полями: `name`, `photoUrls`                                                             | 200                               | 200<br><br>API генерирует одинаковые id для разных объектов, созданный объекты не может быть получен по этому id так как перезапишется другим объектом          |
| Проверить создание питомца гетом                                                                                        | 200, питомец создан               | 200 <br><br>как и ожидалось, при передаче id питомец гарантировано может быть идентифицированным, в отличии от первого теста когда присваивался один и от же id |
| Создать питомца со всеми полями: `id`, `category`, `name`, `photoUrls`, `tags`, `status` передавая поля послеловательно | 200, все переданные поля в ответе | 200, все переданные поля в ответе<br><br>                                                                                                                       |

### Negative

| Тест                                       | Ожидаемый результат | Фактический результат                                                            |
| ------------------------------------------ | ------------------- | -------------------------------------------------------------------------------- |
| `name` = пустая строка `""`                | 405                 | 200, питомец создан                                                              |
| обязательное поле `name` отсутствует       | 405                 | 200, питомец создан                                                              |
| обязательное поле  `photoUrls` отсутствует | 405                 | 200, питомец создан                                                              |
| `photoUrls` содержит ссылки не на фото     | 405                 | 200, питомец создан                                                              |
| невалидное значение `status = "DOG"`       | 405                 | 200, питомец создан                                                              |
| `id`  отрицательное число `-1`             | 405                 | 200, питомец создан с дефолтным id (тот самый одинаковые id для разных объектов) |
| `id` строка `"abc"`                        | 405                 | 500                                                                              |
| `id` дробное число `3.5`                   | 400                 | 200, id преобразован в целое                                                     |
| `id` очень большое число                   | 400                 | 500                                                                              |
| `name` длинная строка (8000 символов)      | 405                 | 200, питомец создан                                                              |
| `name` содержит спецсимволы `G@W[mynpw`    | 405                 | 200, питомец создан                                                              |
| Тело запроса пустое                        | 405                 | 200, создана запись с дефолтными значениями                                      |
| Создать питомца с уже существующим `id`    | 405                 | 200, данные перезаписаны (мы потеряли питомца который был ранее под этим id)     |

## GET /pet/{petId} (получение питомца по ID)

###  Positive

Уже делали, когда проверяли записался ли созданный питомец в базу

### Negative

| Тест                           | Ожидаемый результат     | Фактический результат              |
| ------------------------------ | ----------------------- | ---------------------------------- |
| `petId` строка `"abc"`         | 400 Invalid ID supplied | 404 + NumberFormatException в теле |
| `petId` отрицательный `-1`     | 400 Invalid ID supplied | 404 Pet not found                  |
| `petId`  очень большое число   | 400                     | 404 + NumberFormatException в теле |
| `petId` несуществующий         | 404 Pet not found       | 404 Pet not found                  |
| `petId` с ведущими нулями `09` | 400                     | 200, id преобразован в `9`         |

## GET /pet/findByStatus (поиск по статусу)

### Positive

| Тест               | Ожидаемый результат                        | Фактический результат                      |
| ------------------ | ------------------------------------------ | ------------------------------------------ |
| `status=available` | 200, массив питомцев со статусом available | 200, массив питомцев со статусом available |
| `status=pending`   | 200, все питомцы имеют status=pending      | 200, все питомцы имеют status=pending      |
| `status=sold`      | 200, все питомцы имеют status=sold         | 200, все питомцы имеют status=sold         |

### Negative

| Тест                              | Ожидаемый результат  | Фактический результат                                                       |
| --------------------------------- | -------------------- | --------------------------------------------------------------------------- |
| `status="DOG"` (невалидный)       | 400                  | 200 с данными                                                               |
| `status` не передан               | 400                  | 200 с пустым результатом, несмотря на то что параметр является обязательным |
| `status="SOLD"` (верхний регистр) | 400 или нормализация | 200 с данными                                                               |

## GET /pet/findByTags (поиск по тегам)

### Positive

Перед тестом создать питомцев с уникальными тегами

| Тест                               | Ожидаемый результат                                  | Фактический результат                     |
| ---------------------------------- | ---------------------------------------------------- | ----------------------------------------- |
| `tags=tag1`, один тег              | 200, массив питомцев с tag1                          | 200                                       |
| `tags=tag1, tag2`, несколько тегов | 200, объединённый результат питомцев с tag1 или tag2 | 200, учитывается только первый тег (tag1) |

### Negative

| Тест                      | Ожидаемый результат   | Фактический результат                                                       |
| ------------------------- | --------------------- | --------------------------------------------------------------------------- |
| `tags` несуществующий тег | 200 с пустым массивом | 200 с пустым массивом                                                       |
| `tags` не передан         | 400                   | 200 с пустым результатом, несмотря на то что параметр является обязательным |

## PUT /pet (обновление питомца)

### Positive

| Тест                                      | Ожидаемый результат               | Фактический результат |
| ----------------------------------------- | --------------------------------- | --------------------- |
| Обновить `name` существующего питомца     | 200, `name` изменён               | 200                   |
| Добавить `category` существующему питомцу | 200, `category` добавлена         | 200                   |
| Добавить `tags` существующему питомцу     | 200, `tags` добавлены             | 200                   |
| Добавить `status` существующему питомцу   | 200, `status` добавлен            | 200                   |
| Непереданные поля удаляются               | 200, поля удалены (полная замена) | 200                   |

### Negative

| Тест                                       | Ожидаемый результат | Фактический результат             |
| ------------------------------------------ | ------------------- | --------------------------------- |
| Обязательное поле `name` отсутствует       | 405                 | 200, поле удалено                 |
| Обязательное поле  `photoUrls` отсутствует | 405                 | 200, поле удалено                 |
| `id` несуществующего питомца               | 404                 | 200, создана новая сущность       |
| `status` = невалидный `"ПЕС"`              | 405                 | 200                               |
| Тело пустое                                | 405                 | 200,обновлена дефолтная сущность  |
| `id` не передан                            | 405                 | 200, обновлена дефолтная сущность |
| `id = -98` (отрицательный)                 | 400                 | 200, обновлена дефолтная сущность |
| `id = -08` (отрицательный с ведущим нулём) | 400                 | 400                               |


## POST /pet/{petId} (обновление через form data)

### Positive

| Тест                                    | Ожидаемый результат  | Фактический результат                                                  |
| --------------------------------------- | -------------------- | ---------------------------------------------------------------------- |
| Обновить `name`                         | 200, имя обновлено   | 200, ответ не содержит обновлённые данные, но имя питомца обновился    |
| Обновить `status`                       | 200, статус обновлен | 200, ответ не содержит обновлённые данные, но статус питомца обновился |
| Обновить `name` и `status` одновременно | 200                  | 200, ответ не содержит обновлённые данные, но данные обновились        |

### Negative

| Тест                                | Ожидаемый результат | Фактический результат         |
| ----------------------------------- | ------------------- | ----------------------------- |
| Пустые значения `name` и `status`   | 405                 | 200, изменения не применяются |
| `status = "статус"`<br>(невалидный) | 405                 | 200, сохранён                 |
| `petId` несуществующего питомца     | 404                 | 404                           |
| `name = " "` <br>пробельная строка  | 405                 | 200, сохранён                 |
| `name` содержит спецсимволы         | 405                 | 200, сохранён                 |
| `petId = "pet"`<br>строка           | 400                 | 404 + NumberFormatException   |

## POST /pet/{petId}/uploadImage (загрузка изображения)

### Positive

| Тест                                  | Ожидаемый результат         | Фактический результат           |
| ------------------------------------- | --------------------------- | ------------------------------- |
| Загрузить валидный PNG                | 200, message с именем файла | 200, `photoUrls` не обновляется |
| Загрузить валидный JPEG               | 200                         | 200, `photoUrls` не обновляется |
| Загрузить файл с `additionalMetadata` | 200, metadata в ответе      | 200, `photoUrls` не обновляется |

### Negative

| Тест                                      | Ожидаемый результат | Фактический результат                         |
| ----------------------------------------- | ------------------- | --------------------------------------------- |

| Поле `file` отсутствует (только metadata) | 405                 | 500 Internal Server Error                     |
| Загрузить `.py` файл вместо изображения   | 405                 | 200, тип файла не валидируется, файл загружен |
| Файл 0 байт                               | 405                 | 200, `./null, 0 bytes`                        |
| Файл 100 MB                               | 405                 | Timeout, нет ответа                           |
| Пустая форма                              | 405                 | 200, `./null, 0 bytes`                        |
